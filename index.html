Regression vs Classification

Every supervised ML problem falls into one of two categories. Understanding this distinction is critical — it determines your choice of model, loss function, and evaluation metric before you write a single line of code.

📈
📈

Regression

Predicts a continuous numeric value
  • → Output: any real number $\hat{y} \in \mathbb{R}$
  • → Loss: MSE, MAE, Huber
  • → Metrics: R², RMSE, MAE, MAPE
  • → Final layer: linear / no activation
Example output
Predict house price: $\hat{y} = \$4{,}73{,}000$
Common algorithms
Linear Reg.
Ridge / Lasso
SVR
XGBRegressor

🔀

Classification

Predicts a discrete class label
  • → Output: class label or probability $\hat{p} \in [0,1]$
  • → Loss: Cross-Entropy, Hinge
  • → Metrics: Accuracy, F1, AUC-ROC, Log Loss
  • → Final layer: Sigmoid (binary) / Softmax (multi)
Example output
Predict fraud: $\hat{p} = 0.93 \Rightarrow$ Class = Fraud ✓
Common algorithms
Logistic Reg.
KNN
Decision Tree

📉 Regression Loss Functions

MSE — Mean Squared Error
$\mathcal{L}_{MSE} = \dfrac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2$
MAE — Mean Absolute Error
$\mathcal{L}_{MAE} = \dfrac{1}{n}\sum_{i=1}^{n}|y_i - \hat{y}_i|$
Huber Loss (robust to outliers)
$\mathcal{L}_{\delta} = \begin{cases}\frac{1}{2}(y-\hat{y})^2 & |y-\hat{y}|\le\delta \\ \delta\bigl(|y-\hat{y}|-\frac{\delta}{2}\bigr) & \text{otherwise}\end{cases}$

📉 Classification Loss Functions

Binary Cross-Entropy
$\mathcal{L}_{BCE} = -\dfrac{1}{n}\sum_{i}\bigl[y_i\log\hat{p}_i + (1-y_i)\log(1-\hat{p}_i)\bigr]$
Categorical Cross-Entropy
$\mathcal{L}_{CCE} = -\sum_{i}\sum_{k} y_{ik}\log\hat{p}_{ik}$
Hinge Loss (SVM)
$\mathcal{L}_{hinge} = \dfrac{1}{n}\sum_{i}\max\bigl(0,\, 1 - y_i\hat{f}(x_i)\bigr)$
🧭

Quick Decision Guide — Which Type Is My Problem?

📈 Use Regression when…

  • ✓ Target is a real number (price, score, count)
  • ✓ You care about magnitude of error
  • ✓ Answer question: "How much?" or "How many?"

🔀 Use Classification when…

  • ✓ Target belongs to a finite set of classes
  • ✓ You need a decision / label
  • ✓ Answer question: "Which category?" or "Yes or no?"
💡 Pro tip: Logistic Regression, Decision Trees, KNN, XGBoost and LightGBM all have both a Regressor and a Classifier variant — the algorithm is the same; what changes is the loss function and output activation.

01 · SUPERVISED LEARNING · REGRESSION

Linear Regression

Linear Regression is the foundation of predictive modeling. It models the relationship between one or more input features and a continuous target variable by fitting a straight line (or hyperplane) to the data.

Core Idea

Model Equation
$\hat{y} = \beta_0 + \beta_1 x_1 + \beta_2 x_2 + \cdots + \beta_n x_n$
In Matrix Form
$\hat{y} = X\beta \quad \Rightarrow \quad \hat{\beta} = (X^T X)^{-1} X^T y$
  • → $\beta_0$ is the intercept (bias)
  • → $\beta_i$ are the feature weights (slopes)
  • → $(X^TX)^{-1}X^Ty$ is the closed-form Normal Equation solution

Loss Function — MSE

Mean Squared Error (objective to minimize)
$\mathcal{L} = \frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2$
Gradient Descent Update
$\beta \leftarrow \beta - \alpha \cdot \frac{\partial \mathcal{L}}{\partial \beta}$
  • → $\alpha$ = learning rate
  • → Convex loss → single global minimum
  • → GD or closed-form solution both valid

Key Assumptions

  • ✓ Linearity — relationship between inputs and output is linear
  • ✓ Independence — observations are independent
  • ✓ Constant variance — errors have equal spread (no pattern)
  • ✓ Normality — errors follow a normal distribution
  • ✓ No multicollinearity — features are not highly correlated

Regularization Variants

Ridge L2 Penalty $\lambda\|\beta\|^2$
Lasso L1 Penalty $\lambda\|\beta\|_1$
ElasticNet L1 + L2 combined

Evaluation Metrics

R² (Coefficient of Determination)
$R^2 = 1 - \dfrac{\sum (y_i - \hat{y}_i)^2}{\sum (y_i - \bar{y})^2}$
Numerator -> error from model
Denominator -> error from predicting mean
MAE (Mean Absolute Error)
$\text{MAE} = \dfrac{1}{n}\sum|y_i - \hat{y}_i|$
RMSE
$\text{RMSE} = \sqrt{\dfrac{1}{n}\sum(y_i - \hat{y}_i)^2}$

02 · SUPERVISED LEARNING · CLASSIFICATION

Logistic Regression

Despite its name, Logistic Regression is a classification algorithm. It uses the sigmoid function to squash the linear output into a probability $[0, 1]$, making it ideal for binary (and with softmax, multi-class) classification tasks.

Logistic Regression Equation

Step 1: Linear combination
$z = w_1x_1 + w_2x_2 + \cdots + w_nx_n + b$
Step 2: Apply Sigmoid Function
$\hat{y} = \sigma(z) = \dfrac{1}{1 + e^{-z}}$
Combined Form
$\hat{y} = \dfrac{1}{1 + e^{-(w_1x_1 + w_2x_2 + \cdots + w_nx_n + b)}}$
Interpretation
Output $\hat{y}$ = probability (0 to 1).
If $\hat{y} > 0.5$ -> class 1.
Else -> class 0.

Decision & Multi-class

Multiclass Logistic Regression Equation
Step 1: Linear scores for each class
$z_k = w_k^T x + b_k$
Step 2: Apply Softmax
$P(y=k|x) = \dfrac{e^{z_k}}{\sum_{j=1}^{K} e^{z_j}}$
What it means
Converts scores into probabilities across all classes.
All probabilities sum to 1.
Prediction
$\hat{y} = \arg\max_k P(y=k|x)$
Choose class with highest probability.

Loss — Cross-Entropy

Binary Cross-Entropy (Log Loss)
$\mathcal{L} = -\dfrac{1}{n}\sum_{i}\bigl[y_i\log\hat{p}_i + (1-y_i)\log(1-\hat{p}_i)\bigr]$
Categorical Cross-Entropy (Multi-class)
$\mathcal{L} = -\dfrac{1}{n}\sum_{i=1}^{n}\sum_{k=1}^{K} y_{ik}\log\hat{p}_{ik}$
  • ✓ No closed-form — solved via Gradient Descent / BFGS
  • ✓ Loss is convex → guaranteed to find global minimum
  • ✓ L1/L2 regularization available (penalty parameter C)
  • ✓ Outputs are interpretable probabilities

03 · INSTANCE-BASED LEARNING

K-Nearest Neighbors (KNN)

KNN is a non-parametric, lazy learning algorithm. It makes predictions by finding the K closest training examples to a query point and aggregating their labels. No explicit training phase — the model IS the dataset.

Distance Metrics

Euclidean Distance (most common)
$d(p,q) = \sqrt{\sum_{i=1}^{n}(p_i - q_i)^2}$
Manhattan Distance
$d(p,q) = \sum_{i=1}^{n}|p_i - q_i|$
KNN in brief (steps)
  • 1 Choose the value of K.
  • 2 Compute distance from query point to all training points.
  • 3 Sort distances and select the K nearest neighbors.
  • 4 Predict using majority vote (classification) or mean (regression).

Prediction Rules

  • ✓ Classification — majority vote among K neighbors
  • ✓ Regression — mean (or weighted mean) of K neighbors
  • ✓ K=1 — very flexible, high variance, low bias
  • ✓ Large K — smoother boundary, high bias, low variance
  • ⚠ Features must be scaled (StandardScaler / MinMax)
  • ⚠ Suffers from the curse of dimensionality
Optimal K via
Cross-Validation on validation set, often $K = \sqrt{n}$ as starting point

Pros & Cons

✓ Pros

  • + Simple, no training time
  • + Naturally handles multi-class
  • + Non-linear decision boundaries
  • + No model assumptions

✗ Cons

  • − Slow inference — O(n·d)
  • − Large memory footprint
  • − Sensitive to irrelevant features
  • − Works poorly in high dimensions

Bias-Variance Tradeoff

Expected Test Error
$\text{Error} = \text{Bias}^2 + \text{Variance} + \text{Noise}$
  • → High Bias: model is too simple, misses true pattern.
  • → High Variance: model is too complex, fits noise.
  • ✓ Best models balance both for low test error.

Overfitting vs Underfitting

  • ⚠ Underfitting: high train error + high validation error.
  • ⚠ Overfitting: low train error + high validation error.
  • ✓ Good fit: low train and validation errors, small gap.
How to Fix
  • + Underfitting → increase model capacity, add features, reduce regularization.
  • + Overfitting → regularize, simplify model, get more data, early stop.

Decision Tree

Decision Trees recursively partition the feature space by selecting the split that maximizes information gain (classification) or minimizes variance (regression). They are the fundamental building block of ensemble methods.

Key Components

Root Node

The top node that picks the best feature to split the data first.

Internal Nodes

Decision points that apply feature-based rules and split records.

Leaf Nodes

Terminal outputs: class label for classification or value for regression.

Branches

Connections showing outcomes of each split condition.

How Decision Trees Work

The process of creating a decision tree involves:

1
Selecting the Best Attribute

Use Gini impurity, entropy, or information gain to choose the most useful split.

2
Splitting the Dataset

Partition the data into subsets according to the selected feature rule.

3
Repeating the Process

Repeat recursively for each subset until stopping criteria are met (pure node or max depth).

4
Assign Each Leaf Node Prediction

Majority class (classification) or mean target (regression).

Common Stopping Criteria
  • ✓ Node is pure (all samples same class).
  • ✓ Max depth reached.
  • ✓ Min samples split/leaf constraints triggered.

Entropy, IG & Gini

Entropy (uncertainty / disorder)
$\text{Entropy}(S) = -\sum_{k=1}^{K} p_k \log_2 p_k$
Information Gain (depends on entropy)
$\text{IG} = \text{Entropy(parent)} - \sum_j \frac{n_j}{n}\text{Entropy(child}_j\text{)}$
  • → Entropy measures uncertainty or disorder in a node.
  • → Low entropy = more ordered/homogeneous set; high entropy = more disorder/diversity.
  • → Decision trees reduce entropy by choosing better features/split points.
  • → The split with the largest entropy reduction (highest IG) is selected.
Gini Impurity (separate criterion)
$\text{Gini}(S) = 1 - \sum_{k=1}^{K} p_k^2$
  • → Gini impurity measures impurity/disorder of elements in a node.
  • → Lower Gini means a more homogeneous node and a better split.
  • → CART-style trees choose feature and split point that minimize Gini at each node.
Decision Tree Regression (what is used?)
In regression trees, neither Gini nor Entropy is used. Instead, splits are chosen by minimizing error/variance.
Mean Squared Error (MSE) — most common
$\text{MSE} = \dfrac{1}{n}\sum_{i=1}^{n}(y_i - \bar{y})^2$
  • → Measures how far target values are from the node mean.
  • → Goal: choose split that minimizes post-split MSE.
How splitting works
  • 1 Try candidate split points.
  • 2 Compute MSE of left and right child nodes.
  • 3 Compute weighted total error: $(n_L/n)\,\text{MSE}_L + (n_R/n)\,\text{MSE}_R$.
  • 4 Pick the split with the lowest weighted error.

Decision Tree — Key Limitations

  • ⚠️ Overfitting: Deep trees can memorize training data and generalize poorly.
  • ⚠️ High variance: Small data changes can produce a very different tree.
  • ⚠️ Greedy splits: Best local split at each node may miss the global optimum.
  • ⚠️ Feature bias: High-cardinality categorical features can dominate Information Gain.
  • ⚠️ Linear pattern limits: Piecewise constant leaves struggle with smooth trends.
  • ⚠️ Depth cost: Very large trees increase comparison steps at inference time.
  • ⚠️ Axis-aligned splits: Diagonal boundaries usually require deeper trees.
  • ⚠️ Small-data risk: Trees can fit noise when training samples are limited.
  • ⚠️ Regression cap: Leaf means are constant and cannot extrapolate beyond training range.

Bagging & Boosting

Ensemble methods combine many weak learners into a strong model. Bagging builds models in parallel on bootstrapped samples to reduce variance. Boosting builds models sequentially, each correcting the errors of the last.

Trees trained in parallel on different bootstrap samples — predictions averaged / majority-voted to reduce variance

How it works

  1. 1
    Bootstrap Sampling

    Draw $B$ random samples (with replacement) of size $n$ from training set

  2. 2
    Train Independent Models

    Fit a base learner (e.g., decision tree) on each bootstrap sample independently

  3. 3
    Aggregate Predictions

    Classification: majority vote · Regression: average of all $B$ predictions

Aggregated Prediction
$\hat{f}(x) = \dfrac{1}{B}\sum_{b=1}^{B} \hat{f}^{(b)}(x)$

Random Forest — Bagging Extended

  • → Adds feature randomness: each split considers only $\sqrt{p}$ features
  • → Decorrelates trees → further reduces variance
  • → Out-of-Bag (OOB) score: free validation using unused ~37% samples
  • → Feature importance via mean impurity decrease (MDI)

Sequential training — each learner corrects the previous one's errors by up-weighting misclassified samples

Core Principle (AdaBoost)

  1. 1
    Initialise Weights

    Each training sample gets equal weight $w_i = \frac{1}{n}$

  2. 2
    Train Weak Learner

    Fit a stump/tree on weighted data; compute weighted error $\epsilon_t$

  3. 3
    Compute Learner Weight

    $\alpha_t = \tfrac{1}{2}\ln\!\left(\tfrac{1-\epsilon_t}{\epsilon_t}\right)$ — better models get more say

  4. 4
    Update Sample Weights

    Misclassified samples get higher weights; repeat

  5. 5
    Final Prediction

    $\hat{y} = \text{sign}\!\left(\sum_t \alpha_t h_t(x)\right)$

Gradient Boosting Framework

  • → Generalises boosting to any differentiable loss function
  • → Each tree fits the pseudo-residuals (negative gradient of loss)
Additive Update Rule
$F_m(x) = F_{m-1}(x) + \nu \cdot h_m(x)$
where $\nu$ = learning rate, $h_m$ = tree fitted on residuals

learning_rate ν Shrinks each tree's contribution
max_depth Shallow trees (1–5) typical

06 · GRADIENT BOOSTING · IN DEPTH

XGBoost

eXtreme Gradient Boosting — Chen & Guestrin, 2016
Industry Standard

XGBoost is gradient boosting on steroids, with three big upgrades: regularization, second-order optimization, and systems-level engineering.

🎯 The Analogy

"XGBoost is like a careful, disciplined coach. Each session (tree) is short and focused. The coach doesn't push the student too hard (ν, the learning rate) and refuses to add a drill unless it clearly helps (the Gain check with γ). They also won't let the student over-specialise on one weakness (the L2 penalty λ). The result is a student who's strong and well-rounded."

Shallow trees (ν controls step size)

No Over-Specialising
L2 penalty λ keeps weights balanced

⚙️ Key Innovations

1
Sparsity-Aware Split Finding
Handles missing values intelligently

🎯 The XGBoost Objective

XGBoost minimises a regularised objective, adding a complexity penalty to the standard loss:

$\mathcal{L}^{(t)} = \sum_{i=1}^{n}\ell\!\left(y_i,\, \hat{y}_i^{(t-1)} + f_t(x_i)\right) + \Omega(f_t)$
Regularisation term
$\Omega(f) = \gamma T + \dfrac{1}{2}\lambda\sum_{j=1}^{T}w_j^2$
$T$ = number of leaves $w_j$ = leaf weight (score) $\gamma$ = min gain to split $\lambda$ = L2 weight regularisation

Taylor Expansion Trick (2nd Order)

Unlike vanilla GBM that uses only 1st-order gradient, XGBoost uses both gradient $g_i$ and Hessian $h_i$:

1st-order gradient
$g_i = \partial_{\hat{y}^{(t-1)}}\ell(y_i, \hat{y}^{(t-1)})$
2nd-order Hessian
$h_i = \partial^2_{\hat{y}^{(t-1)}}\ell(y_i, \hat{y}^{(t-1)})$
Optimal leaf weight (closed-form)
$w_j^* = -\dfrac{\sum_{i \in I_j} g_i}{\sum_{i \in I_j} h_i + \lambda}$
Split Gain
$\text{Gain} = \dfrac{1}{2}\!\left[\dfrac{G_L^2}{H_L+\lambda} + \dfrac{G_R^2}{H_R+\lambda} - \dfrac{(G_L+G_R)^2}{H_L+H_R+\lambda}\right] - \gamma$

📊

GOSS — Gradient-based One-Side Sampling

Not all data samples are equally informative. GOSS keeps samples with large gradients (under-trained instances) and randomly discards a proportion of small-gradient samples.

  1. 1
    Sort by |gradient| descending

    Find the top $a\%$ large-gradient instances → always kept

  2. 2
    Random sample from remaining

    Randomly pick $b\%$ from the small-gradient instances

  3. 3
    Amplify small-gradient contribution

    Multiply small-gradient samples by $\frac{1-a}{b}$ factor to correct for distribution

Amplification factor
$\tilde{g}_i = \dfrac{1-a}{b} \cdot g_i \quad \text{for small-gradient samples}$

🗜️

EFB — Exclusive Feature Bundling

In high-dimensional sparse data, many features are mutually exclusive — they rarely take non-zero values simultaneously. EFB bundles these into single features without information loss.

  1. 1
    Build Feature Conflict Graph

    Two features "conflict" if they are simultaneously non-zero more than $\gamma$ fraction of samples

  2. 2
    Greedy Bundling via Graph Colouring

    Group features with low conflicts — NP-hard problem approximated greedily

  3. 3
    Merge into One Feature

    Offset values to distinguish origin — no information lost in the histogram

📌 Net effect: Reduces #features from $D$ to $K \ll D$, drastically cutting histogram construction time

🌱 Leaf-wise vs Level-wise Growth

❌ Level-wise (XGBoost default)

  • → Grows all leaves at the same depth
  • → Safer — lower risk of overfitting
  • → Often wastes splits on low-gain leaves

✅ Leaf-wise (LightGBM)

  • → Always expands the leaf with max delta loss
  • → Achieves lower loss with fewer iterations
  • → More prone to overfit on small datasets — use num_leaves / min_data_in_leaf
